## Définitions

### Definition of Ready (DoR)

Chaque User Story doit :

* [ ] Être comprise
* [ ] Être estimée
* [ ] Avoir ses critères d'acceptation
* [ ] Ne pas avoir de dépendances bloquantes

### Definition of Done (DoD)

Chaque fonctionnalité doit :

* [ ] Compiler
* [ ] Être testée
* [ ] Être commitée sur GitHub
* [ ] Être documentée
* [ ] Être démontrable

# 💡 Conseils

* Faire des commits Git régulièrement.
* Écrire des messages de commit clairs.
* Tester avant chaque commit important.
* Ne jamais attendre la fin de journée pour fusionner les branches.
* Demander rapidement au Product Owner lorsqu'un besoin est ambigu.
* Prioriser les fonctionnalités du MVP avant de commencer la Release 2.
